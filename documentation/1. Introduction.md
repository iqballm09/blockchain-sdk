# 1. Introduction

### Blockchain 
1. immutable transaction ledger, distributed to all nodes.
2. Each Node maintains a copy of ledger (grouped of blocks).
3. Each transacion has been validated by consensus protocol before grouped into blocks.
4. Each block includes hash that binds each block to preeceding block.

### Permissionless / Public Blockchain
1. Publick networks, open to anyone, participats interact anonymously
2. Bitcoin and Ethereum
3. Cannot being applied to every use case that identity of the participats is hard requirement such as Know-Your-Customer (KYC) and Anti-Money Laundering (AML) regulation.

### Permissioned / Enterprise Blockchain
1. Participats must be identified
2. Networks need to be permissioned
3. Focused on high transaction performance
4. Transaction confirmation with low latency
5. Privary and confidentiality of transactions

### Hyperledger Fabric
1. Open source Distributed Ledger Technology (DLT) platform 
2. Highly Modular and configurable architecture, can be adjusted as needs.
3. Supported general purpose programming language such as java, go, and node.js
4. Permissioned, participants must been authorized and knowing each other
5. Pluggable consensus protocols, can select protocol to fit particular use cases.
6. Protocols: Crash Fault Tolerant (CFT), Byzantine Fault Tolerant (BFT), RAFT
7. Do not require a gas fee

### Modularity
1. Pluggable ordering service
2. pluggable Membership Service Provider (MSP)
3. Smart contract is running through docker
4. Ledger can be supportied DBMSs
5. Pluggable endorsement and validation policy

### Smart Contract / Chaincode
1. Implementation of business logics
2. Execute-order-validate mechanism
- Execute transaction and check its correctness, then endorsing it
- Order transactions via consensus protocol (BFT, CFT, RAFT)
- Validate transactions against an based on endorsement policy, then committing them to ledger.

### Privacy & Confidentiality
1. Channeling Architecture & Private data feature
2. In channels, participants in specific channel have sub-network where they have visibility of specific data and having access to smart contract. 
3. Private data allows data collections between members on channel. Same function as channel
